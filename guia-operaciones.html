Guía de Operaciones

Todo lo que necesitás saber para vender, configurar, publicar y mantener landings dentales para tus clientes. Sin conocimientos técnicos avanzados.

🔄 Flujo de trabajo completo

De la venta a la entrega en 4 pasos simples.

1
Vendés el servicio a la clínica

Mostrás la demo (el HTML de la landing con datos de ejemplo). Le explicás que en 48h tienen su página personalizada y funcionando online. Cobrás por adelantado.

2
El cliente completa el formulario de onboarding

Le mandás el archivo onboarding-dental.html. Lo abre en su navegador, completa sus datos en 10 minutos y te manda el bloque CONFIG generado por el formulario.

3
Vos pegás el CONFIG en la landing y la publicás

Abrís clinica-dental-landing.html en un editor de texto, reemplazás el bloque const CONFIG = {'{...}'} con el que te mandó el cliente, y subís el archivo al hosting. Listo.

4
Entregás y cobrás el mantenimiento mensual

Le mandás la URL al cliente. Si contrata mantenimiento, ante cualquier cambio (precio, horario, servicio nuevo) el cliente te avisa, vos lo actualizás en el CONFIG y subís el archivo. Tarda menos de 5 minutos.

📋 Onboarding del cliente

El formulario recopila todo sin que vos tengas que preguntar nada.

📤 Cómo mandarlo

El archivo onboarding-dental.html es standalone — no necesita servidor, se abre directo en el navegador. Opciones para enviarlo:

A
Por email (adjunto)El cliente lo descarga y lo abre. Funciona en Chrome, Edge, Firefox.
B
Por Google Drive / WeTransferLo subís vos y le mandás el link de descarga.
C
Subido a Netlify DropArrastrás el HTML a app.netlify.com/drop y le mandás el link. Queda online permanentemente y el cliente lo puede abrir desde el teléfono.
💡 Tip: Si el cliente no entiende cómo abrir el archivo, grabá un video de Loom de 2 minutos mostrándolo. Lo reutilizás para todos los clientes.

⚙️ Configurar la landing

Una vez que tenés el bloque CONFIG del cliente, el proceso es simple.

🛠️ Qué editor usar

Recomendamos Visual Studio Code (gratis). Si no querés instalar nada, usá Notepad++ (Windows) o el editor de texto del sistema.

Descargá VS Code en code.visualstudio.com — gratis, liviano, y resalta el código en colores.

📝 Paso a paso — reemplazar el CONFIG

1
Abrí el archivo en VS CodeArrastrá clinica-dental-landing.html a VS Code o hacé clic derecho → Abrir con → VS Code.
2
Usá Buscar y Reemplazar Presioná Ctrl+H (Windows) o Cmd+H (Mac).
3
Buscá el bloque CONFIG Buscá el texto /* ── Clínica ── */ — ese es el inicio del bloque que vas a reemplazar. Seleccioná desde ahí hasta el cierre de los mensajes.
4
Pegá el nuevo CONFIG del clienteReemplazá lo seleccionado con el bloque que generó el formulario de onboarding. Guardá con Ctrl+S.
5
Verificá abriendo el archivo en el navegadorDoble clic en el HTML → se abre en Chrome. Revisá que el nombre de la clínica, teléfono y servicios estén correctos.

🔍 También editar en el HTML (textos fijos)

Algunos textos no están en el CONFIG porque son raramente cambiados. Para editarlos, buscá los comentarios ← EDITAR en el HTML con Ctrl+F. Entre ellos:

/* Estos se buscan con Ctrl+F ← EDITAR */ <!-- EDITAR: nombre --> → nombre en el navbar <!-- EDITAR: año --> → año de fundación en el hero schema.org → datos SEO estructurados (dirección, teléfono, horarios) <meta og:image> → imagen para compartir en redes (1200×630px)

🚀 Publicar online

Tres opciones según el presupuesto del cliente. Todas funcionan con un único archivo HTML.

⚡ Opción A — Netlify Drop (gratis, 2 minutos)

Ideal para empezar rápido o para demos. El cliente puede usar su propio dominio.

1
Ir a app.netlify.com/drop
2
Arrastrar el archivo HTMLNetlify genera una URL tipo nombre-random.netlify.app.
3
Conectar dominio propio (opcional)Desde el panel de Netlify → Domain settings → Add custom domain.
✅ Gratis, HTTPS automático, CDN global. Perfecto para la mayoría de los clientes.

🌐 Opción B — GitHub Pages (gratis, requiere cuenta GitHub)

1. Crear repositorio en github.com 2. Subir el HTML renombrado como index.html 3. Settings → Pages → Deploy from branch: main 4. URL: https://usuario.github.io/nombre-repo

🏠 Opción C — Hosting del cliente (cPanel / FTP)

Si el cliente ya tiene hosting (GoDaddy, Hostinger, SiteGround, etc.):

1
Acceder al File Manager del cPanel
2
Navegar a la carpeta public_html
3
Subir el HTML renombrado como index.htmlSi ya existe un index.html viejo, hacer backup antes.

❓ Preguntas frecuentes

¿El cliente puede editar la landing solo?

No directamente — necesitan editar código HTML. Por eso existe el modelo de mantenimiento mensual: vos hacés los cambios por ellos. Si el cliente insiste en editarlo solo, podés usar el admin-panel.html que genera el CONFIG, y ellos te lo mandan a vos para actualizar.

¿El chatbot envía citas reales a algún sistema?

Sí: cuando el paciente confirma, se abre WhatsApp Business con el resumen completo de la cita (o un mailto si se configura con "email"). No hay integración con calendarios automáticos — eso requeriría un backend. Para la mayoría de las clínicas pequeñas, el WhatsApp es suficiente.

¿Se puede cambiar el color o la tipografía?

Sí. Los colores están en el bloque :root al inicio del CSS. Cambiás --teal, --navy y --gold y toda la página se adapta. Las fuentes están en el link de Google Fonts en el head.

¿Funciona en todos los navegadores?

Sí — Chrome, Firefox, Safari, Edge. Compatible con mobile. No funciona en Internet Explorer (pero IE tiene menos del 0.5% de uso global en 2025).

¿Necesito un servidor o base de datos?

No. Es un único archivo HTML estático. Funciona en cualquier hosting, incluso los más básicos. Netlify lo hostea gratis.
